**Mòdul 0612 — Desenvolupament Web en Entorn Client (DWEC)**
**Cicle Formatiu de Grau Superior en Desenvolupament d'Aplicacions Web (DAW)**

## En què consisteix este projecte?

Benvolguda, benvolgut alumnat!

A la situació d'aprenentatge **SAComercLocal** construireu una **aplicació web completa i funcional** amb un propòsit real: **donar visibilitat al comerç local del vostre municipi** i contribuir a la seua dinamització. En lloc de fer exercicis aïllats, cada cosa que aprendreu tindrà un lloc on aplicar-la dins d'un projecte amb sentit.

### Què construireu?

Una plataforma digital pensada per apropar els comerços de la vostra localitat a la ciutadania. Una web on qualsevol persona podrà:

- **Navegar** per un catàleg de comerços locals
- **Consultar** les fitxes detallades de cada establiment (horaris, telèfon, descripció, productes...)
- **Cercar i filtrar** comerços per categories
- **Contactar** amb els establiments mitjançant formularis
- **Veure la ubicació** de cada comerç en un mapa interactiu

### Per què val la pena?

Primer, perquè té un **impacte real**: molts xicotets negocis no tenen presència digital, i la web que construireu pot ser la manera que la gent del vostre municipi els descubrisca i els visite.

Segon, i igual d'important, perquè treballareu **exactament com es treballa al sector**: en equip, amb metodologia àgil, gestionant tasques en un tauler Kanban, utilitzant Git per col·laborar en el codi i construint amb React, el framework de JavaScript més demanat al mercat laboral.

## Com treballarem? El nostre procés

El projecte s'organitza en **5 sprints** al llarg de **22 setmanes**, utilitzant la metodologia **Scrum** adaptada a l'aula. Cadascun dels sprints té un objectiu concret i un producte lliurable que anirà construint la web pas a pas.

<img src="https://github.com/RLGARAY/SAComercLocal/blob/main/Professorat/Planificacio_temporal/infografia.png" alt="Infografia de les fases i temporalització del projecte" width="50%" height="50%">

### Com funciona cada sprint?

Cadascun dels sprints segueix una dinàmica fixa que anireu interioritzant:

1. **Sprint Planning** (inici): El professorat presenta les activitats i tasques del sprint. L'equip les revisa, les entén, les distribueix entre els membres i cada membre assumeix un rol.

2. **Standup diari** (cada sessió): Reunió breu de 5-10 minuts on cadascun respon:
   - Què vaig fer l'última sessió?
   - Què faré avui?
   - Tinc algun bloqueig?

3. **Desenvolupament**: Programació, col·laboració i resolució de problemes durant les sessions de classe.

4. **Sprint Review** (final): Demostració del que heu construit, on rebreu feedback del professorat. Seguida d'una retrospectiva on l'equip reflexiona sobre què ha funcionat i què cal millorar.

### Rols rotatius

En cada sprint, els membres de l'equip assumeixen rols diferents que van canviant:

| Rol | Funció |
|-----|--------|
| **Scrum Master** | Coordina l'equip, facilita les reunions de Scrum i és l'enllaç amb el professorat |
| **UI (Disseny)** | Cuida l'aspecte visual i l'experiència d'usuari de la web |
| **Documentador** | Documenta les tasques desenvolupades i les decisions preses |

### Eines de treball

| **VS Code** | Editor de codi on programareu |

## Què avaluaré? Resultats d'Aprenentatge i Criteris

El projecte està vinculat als **Resultats d'Aprenentatge (RA)** del mòdul de Desenvolupament Web en Entorn Client. A continuació trobareu un resum dels RA que treballarem i els criteris d'avaluació associats a cadascun.

### RA1 — Anàlisi d'una aplicació web en entorn client

> Analitzareu el funcionament d'una web, identificant els models d'execució (client/servidor), les capacitats del navegador, els llenguatges de programació de clients web i les particularitats de la programació de scripts.

**Criteris que treballareu:**
- Diferenciar execució en client i servidor
- Identificar les capacitats dels navegadors web
- Identificar els principals llenguatges de programació de clients web
- Reconèixer les particularitats de la programació de scripts
- Verificar mecanismes d'integració de llenguatges de marques i programació
- Reconèixer i avaluar eines de programació i prova sobre clients web

### RA2 — Programació en JavaScript

> Programareu aplicacions utilitzant variables, operadors, estructures de control, funcions i comentaris al codi, seleccionant el llenguatge adequat i utilitzant eines de desenvolupament.

**Criteris que treballareu:**
- Seleccionar un llenguatge de programació adequat
- Utilitzar variables, operadors i àmbits
- Reconèixer conversions de tipus
- Utilitzar mecanismes de decisió i bucles
- Afegir comentaris al codi
- Utilitzar eines de depuració i prova

### RA3 — Objectes predefinits del llenguatge

> Utilitzareu objectes predefinits de JavaScript per interactuar amb el navegador: canviar l'aspecte, generar contingut, interactuar amb l'usuari i emmagatzemar informació.

**Criteris que treballareu:**
- Identificar objectes predefinits del llenguatge
- Analitzar objectes del navegador i documents web
- Canviar l'aspecte del navegador i el document
- Generar textos i etiquetes des de codi
- Interactuar amb l'usuari mitjançant objectes predefinits
- Utilitzar mecanismes d'emmagatzematge al navegador
- Depurar i documentar el codi

### RA4 — Orientació a objectes i funcions

> Creareu funcions pròpies, arrays, objectes i operacions agregades per gestionar la informació de manera estructurada.

**Criteris que treballareu:**
- Utilitzar funcions predefinides del llenguatge
- Definir i utilitzar funcions pròpies
- Crear i utilitzar arrays
- Realitzar operacions agregades (filter, sort, map)
- Aplicar orientació a objectes
- Definir estructures d'objectes, mètodes i propietats
- Aplicar patrons de disseny de programari
- Depurar i documentar el codi

### RA5 — Gestió d'esdeveniments i formularis

> Implementareu esdeveniments, formularis validats i expressions regulars per a millorar la interacció amb l'usuari.

**Criteris que treballareu:**
- Reconèixer les possibilitats de captura d'esdeveniments
- Identificar les característiques de gestió d'esdeveniments
- Diferenciar els tipus d'esdeveniments
- Crear codi que capture i utilitze esdeveniments
- Reconèixer capacitats relatives a formularis web
- Validar formularis amb esdeveniments
- Utilitzar expressions regulars per a la validació
- Provar i documentar el codi

### RA6 — Model d'objectes del document (DOM)

> Manipulareu l'estructura d'una pàgina web de manera dinàmica, accedint, creant i modificant elements del DOM.

**Criteris que treballareu:**
- Reconèixer el model d'objectes del document
- Identificar objectes, propietats i mètodes del DOM
- Crear codi que accedisca a l'estructura del document
- Crear nous elements i modificar existents
- Associar accions als esdeveniments del model
- Independitzar les capes d'implementació (contingut, aspecte, comportament)

### RA7 — Comunicació asíncrona

> Implementareu mecanismes de comunicació asíncrona entre client i servidor, utilitzant la Fetch API i formats JSON.

**Criteris que treballareu:**
- Avaluar avantatges i inconvenients de mecanismes asíncrons
- Analitzar mecanismes disponibles per a la comunicació asíncrona
- Utilitzar objectes relacionats (fetch, Promise, async/await)
- Identificar propietats i mètodes d'aquests objectes
- Utilitzar comunicació asíncrona en l'actualització dinàmica del document
- Utilitzar diferents formats d'enviament i recepció d'informació
- Classificar, analitzar i utilitzar llibreries i frameworks d'actualització dinàmica
- Crear, provar i documentar aplicacions que utilitzen aquestes llibreries

## Com es valora el projecte?

Per a superar el projecte cal obtindre una **puntuació mínima de 5 sobre 10** i no tindre cap criteri amb qualificació d'insuficient.

### Què compta per a la nota?

La nota es calcula a partir de dues fonts principals:

**1. Activitats i entregables de cada sprint**
Al llarg dels 5 sprints anireu realitzant **activitats vinculades a les tasques del tauler Trello**. Cada activitat o entregable té associats un o diversos criteris d'avaluació dels RA del mòdul. Aquestes activitats són les que determinen la qualificació de cadascun dels criteris.

**2. Entrega final del projecte**
Al final del projecte lliurareu la **web completa i funcional**, que és el producte integrador de tot el que heu construït durant els sprints.

### Procés de treball

A més del producte, es farà un seguiment del **procés de treball dels equips**: l'ús correcte del tauler Kanban, la participació en les cerimònies Scrum (standups, planning, review) i la col·laboració en equip.
